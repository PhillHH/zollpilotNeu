### Gaps / Notes

1. **Keine Screen-Umbauten**: Bestehende Screens wurden nicht angepasst, nur die Design-Infrastruktur bereitgestellt. Migration der Screens ist für spätere Prompts geplant.

2. **Dark Mode vorbereitet**: Die Dark-Mode-Variablen sind in `tokens.css` definiert (`[data-theme="dark"]`), aber nicht aktiviert. Aktivierung via `<html data-theme="dark">`.

3. **Inter Font**: Die Schriftart Inter ist als Standard definiert, muss aber vom Browser/System geladen werden. Für Production: Google Fonts oder lokale Installation.

4. **Form-Komponenten fehlen**: Input, Select, Checkbox sind nur als Base-Styles definiert, nicht als eigene Primitives. Bei Bedarf erweitern.

5. **Responsive Breakpoints**: Nur `768px` (md) ist in den Base-Styles definiert. Vollständiges Breakpoint-System könnte ergänzt werden.

## Design-Entscheidungen

| Entscheidung | Begründung |
|--------------|------------|
| CSS Custom Properties statt CSS-in-JS | Einfache Austauschbarkeit, keine Build-Dependency |
| Helle Hintergründe | SaaS/GovTech Ästhetik, professionell |
| Grüne Primärfarbe | Vertrauen, Ruhe (GovTech-typisch) |
| styled-jsx für Primitives | Next.js-native, kein zusätzliches Setup |
| Deutsch-first als Props | Einfache Übersetzung bei Internationalisierung |

## Verwendung

```tsx
// Import
import { Button, Card, Badge, Alert } from '@/app/design-system';

// Button-Varianten
<Button variant="primary">Speichern</Button>
<Button variant="secondary">Abbrechen</Button>

// Status-Badge (automatisch Deutsch)
<Badge status="submitted" />  // → "Eingereicht"

// Alert mit deutschem Standard-Titel
<Alert variant="error">Fehlermeldung</Alert>  // Titel: "Fehler"
```
